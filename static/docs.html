Overview

PhotoEditor is a web workbench for classical image denoise and technical quality reporting. Production host: https://editor.herooflegend.com. It does not use generative AI to redraw photos. Filters run through OpenCV (non-local means, bilateral, Gaussian, median, or a hybrid). By default, resolution is preserved.

Access requires sign-in (email invite + password). Each profile has isolated history and library. Admins can invite users and view other profiles in the workspace. Invite mail uses SMTP (e.g. Resend) or Cloudflare Email Sending.

The left panel holds controls. The center shows the image (with zoom). The right panel shows expandable metric categories. Hover any control’s ? in the editor for an ELI5 tip; this page is the full technical reference. Deploy notes: see the repository docs/deployment.md.

Upload & preview

Image upload

ELI5: Drop a picture so the app can look at it and clean it.

Accepts common raster formats (JPEG, PNG, WebP, …) via drag-and-drop or file picker. The server stores a copy under uploads/, runs single-image analysis, and returns geometry, luminance, color, high-frequency, and noise-proxy metrics.

  • Max upload size: 80 MB
  • Decoded to 8-bit RGB for processing
  • Starts an undo-history step labeled “Original upload”

Preview zoom

ELI5: Make the picture bigger or smaller on screen without changing the real file.

Display size is a percentage of native pixels (100% = one screen pixel per image pixel). Fit chooses the largest scale that fits the stage; 1:1 is true native resolution. Images anchor top-left. Wheel zooms toward the cursor; drag pans when content overflows.

Master denoise

Strength (0–100%)

ELI5: How hard the cleaner tries. 0 = almost nothing. 100 = scrub a lot (may also blur real details).

Maps through a mild power curve (t = (strength/100)^1.2) into algorithm parameters: NLM h, bilateral diameter / σ color / σ space, Gaussian σ, median kernel size, and a blend factor toward the original at low strength (hybrid).

Strength Typical behavior
0–15% Very mild; hybrid keeps most original texture
30–55% General-purpose denoise for mild ISO grain
70–100% Aggressive; watch for plastic skin / lost fabric detail

Algorithm

ELI5: Which “cleaning brush” to use. Hybrid is the default all-rounder.

See Algorithms for technical detail on each option.

Algorithms

Hybrid (NLM + bilateral) — default

ELI5: First finds similar patches to average noise away, then gently smooths edges without totally melting them. Mixed with the original so light strength stays natural.

Pipeline: fastNlMeansDenoisingColored → light bilateralFilter → linear blend with the original using blend = t from strength. Best general default for photos.

Non-local means (NLM)

ELI5: Looks around the picture for similar little patches and averages them so random speckles cancel out.

OpenCV fastNlMeansDenoisingColored. Filter strength h (and color hColor) scale with master strength unless you set NLM h manually. Template window default 7, search window 21.

Strengths: good grain reduction on textured areas. Weaknesses: slower on large images; can create watercolor artifacts if h is too high.

Bilateral

ELI5: Blurs flat areas but tries not to smear across sharp edges (like hair against a wall).

OpenCV bilateralFilter(d, sigmaColor, sigmaSpace). Diameter and sigmas grow with strength, or use manual σ color / σ space (0 = auto).

Gaussian blur

ELI5: Simple soft blur everywhere—like smearing a foggy window. Fast, not picky about edges.

Separable Gaussian with σ from strength (or manual Gaussian σ). Kernel size derived from σ. Useful as a baseline, not ideal as a sole denoise for portraits.

Median

ELI5: Replaces each spot with the middle value of its neighbors—great for weird single-pixel “salt and pepper” dots.

OpenCV medianBlur with odd kernel size from strength. Excellent for impulse noise; can stair-step fine lines if overdone.

Category targets (%)

Optional goals: reduce a measured noise/HF proxy by about N%. When any target is > 0, the backend searches strength (coarse 5–100% grid) to approximate the goal(s). 0 means “ignore; use master strength only.”

Laplacian variance reduce

ELI5: How much to turn down the “sparkly fine detail” energy. That sparkle is both grain and tiny real details.

Target: lower Var(Laplacian(luma)) by the given percent. Laplacian variance rises with fine edges and noise. Strong reduction often flattens pores and fabric weave.

Residual std reduce (5×5)

ELI5: After we blur a little, what’s left is mostly tiny freckles of noise. This says how much of that freckling to remove.

Residual = image − 5×5 box blur; metric is std(residual) on Rec.709 luminance. A solid primary denoise target for grain without requiring a reference pair.

Local std mean reduce

ELI5: In every small neighborhood, how bumpy is the picture? Lower this to make neighborhoods calmer.

Mean of local standard deviation in 5×5 windows (from local mean / mean-square via box filters). Captures texture and noise energy together.

Photometric offsets

Applied after spatial denoise, as additive deltas in 8-bit channel space (then clipped to 0–255). They do not remove noise; they shift brightness/color.

Luminance offset

ELI5: Make the whole photo a little brighter or darker.

Adds the same value to R, G, and B (uniform lift/gain in code path).

R / G / B offsets

ELI5: Nudge just red, green, or blue—like warming or cooling the photo a bit.

Independent channel adds. Example: +R / −B warms the image. Range in UI: about −40…+40 levels.

Advanced parameters

JPEG quality

ELI5: How carefully to save the cleaned picture as a JPEG. Higher = bigger file, fewer save-blobs.

Export quality 40–100 for the denoised download / library output. Default 95.

Scale

ELI5: Shrink or grow the pixel size of the result. Usually leave this alone.

Multiplier on width/height after denoise. Only applied when Preserve resolution is unchecked. Uses area interpolation when downscaling, LANCZOS4 when upscaling.

Preserve resolution

ELI5: Keep the same number of pixels as the input. On by default so nothing secretly shrinks.

When checked, scale is forced to 1.0. Recommended for faithful technical comparison.

NLM h (0 = auto)

ELI5: Manual “how strong is NLM?” knob. 0 means “pick from Strength.”

Overrides automatic h / color strength for NLM and hybrid’s NLM stage.

Bilateral σ color / σ space

ELI5: Color σ = how different colors can still mix. Space σ = how far away neighbors count. 0 = auto from Strength.

Manual OpenCV bilateral sigmas. Large σ color flattens more across edges; large σ space pulls from farther pixels.

Gaussian σ

ELI5: How wide the simple blur is. 0 = auto from Strength.

Only used when Algorithm is Gaussian (or as auto param bookkeeping for other modes).

Actions

Apply

ELI5: Press Apply to run the cleaner. An hourglass and progress bar show while it works.

POST /api/denoise with controls JSON + job id or file. The UI shows staged progress (preparing → filter → metrics → done). Returns report + output JPEG, then pushes an undo step and updates the library.

Re-analyze

ELI5: Measure the current picture again without cleaning it.

Re-runs POST /api/analyze on the loaded file; resets comparison report.

Download

ELI5: Save the cleaned photo to your computer.

Fetches the last denoised JPEG for the job (or blob URL when restored from cache).

Reset controls

ELI5: Put all sliders back to factory defaults. Does not delete your image.

Session, history & library

Browser session cache (IndexedDB)

ELI5: The app remembers your work if you refresh or the tab hiccups.

Store name photoeditor: session blobs, undo stack (source/output blobs + labels), and library entries. Autosave ~30s and on unload; restore on load.

Undo history

ELI5: Every clean is a save-point. Undo goes back; History tab jumps to any step.

Max ~40 steps. Header Undo/Redo and shortcuts ⌘/Ctrl+Z, ⌘⇧Z / Ctrl+Y. Clear history drops the stack only.

Image library (repository)

ELI5: A shelf of past cleaned photos with a short diary of what you did.

Mirrored to server folder library/{id}/ with meta.json, history.json, source/output images. Open/delete from the Library tab.

Clear session / history / library / all

ELI5: Throw away saved work when you want a clean slate.

  • Clear session — WIP cache only
  • Clear history — undo stack
  • Clear library — browser + server repository
  • Clear all cache — everything local + library wipe

Metrics explained

Metrics use Rec.709 luminance Y = 0.2126R + 0.7152G + 0.0722B where noted. Comparison metrics align sizes by LANCZOS-resizing source to output if needed.

Geometry / encoding

ELI5: How big the picture is and how it was saved (JPEG? how many dots?).

Width, height, pixel count, aspect, file bytes, format, bit depth, ICC flag, DPI.

Pixel difference

ELI5: How different the cleaned picture is from the original, pixel by pixel.

  • MAE — mean |Δ|
  • RMSE — root mean square error
  • PSNR — 10·log10(255² / MSE); higher ≈ closer to source
  • % pixels over thresholds — how widespread changes are

High-frequency (Laplacian)

ELI5: How much tiny sparkle (grain + fine detail) is in the photo.

3×3 Laplacian on luma; report variance and mean absolute response. Denoise typically reduces these (negative % change).

Noise proxies

ELI5: Guess-ometers for “how noisy,” without needing a perfect clean reference.

  • Residual std (5×5) — std after subtracting box blur
  • Local std mean/median — neighborhood variability maps

Structural similarity (global SSIM-like)

ELI5: Did the big shapes stay the same even if we cleaned freckles of noise?

Global (not windowed) SSIM-style score on luma and RGB. Near 1.0 means structure largely preserved. Not identical to scikit-image multi-scale SSIM.

Processing pipeline (end-to-end)

  1. Decode upload → RGB float64 [0, 255]
  2. Optional downscale for process if max side > 4000 (analysis may still report original meta)
  3. If category targets > 0: search strength to match proxy reductions
  4. Map strength → algorithm params; run hybrid / NLM / bilateral / Gaussian / median
  5. Apply luminance + RGB offsets; clip
  6. Optional scale if preserve_resolution is false
  7. Encode JPEG; compute full compare report vs pre-denoise RGB
  8. Client: undo step + IndexedDB session + library mirror to server
